## 🔥 数据验证最强公式（基于真实爆款数据）

> 以下公式来自真实数据验证，优先于其他公式使用

**公式1：时间承诺 + 工具名 + 门槛消除（最强，7000+点赞验证）**
```
[X分钟] + 上手/装好/入门 + [工具名] + （小白入门/小白也能学会✅）
```
- 验证案例：`3分钟上手酒馆AI（小白入门）` → 7111点赞
- 验证案例：`2分钟装好ClaudeCode|小白也能学会✅` → 3222点赞
- 验证案例：`15分钟Claude Code小白入门`（对标博主）→ 6781点赞
- **神经机制**：时间承诺触发多巴胺预期（"确定3分钟能完成"），括号身份标签触发镜像神经元认同

**公式2：第一人称分享 + 具体方法（2000+点赞验证）**
```
分享我的 + [具体方法名]
```
- 验证案例：`分享我的AppUI设计方法` → 2140点赞
- **神经机制**：第一人称激活镜像神经元，"朋友在分享经历"而非"专家在授课"

**公式3：口语化问句 + 工具/概念（对标博主验证，12000+点赞）**
```
什么是 + [概念]，以及怎么使用？
如何让 + [具体痛点描述]
```
- 验证案例：`什么是Vibe Coding，以及怎么使用？` → 12606点赞
- 验证案例：`如何让AI写的东西没有AI味` → 8609点赞
- **神经机制**：问句触发认知闭合需求，大脑自动进入"寻找答案"模式

**公式4：权威引用 + 具体数字承诺（对标博主验证，32000+点赞）**

[工具名] + 深度实测！+ [具体数字] + 做到/完成/实现 + [结果]
[权威名] + 最新 + [内容类型]，[主观评价]

**公式5：专家访谈解读（跨博主验证，9000+点赞）**

## 情绪钩子库（通用，次要于上方验证公式）

**警觉类钩子（适合揭秘类）：**
- "千万别XX"
- "XX正在毁掉你"

**好奇类钩子（适合科普类）：**
- "没想到XX"
- "如何让XX"（口语化问句，强于"XX的真相"）

**权威借势类钩子（适合信任转移）：**
- "[权威]最新[内容]，[主观评价]"
- "[工具名]深度实测！[具体数字]做到XX"

**教程入门类钩子（适合工具教程）：**
- "[X分钟]上手/入门 + [工具名]（小白入门）"  ← 最强
- "[工具名]保姆级教程"

## 封面配文规则

**主标题（第一行）：**

**字数要求：** 3-8字

**结构公式（根据驱动策略）：**

**当使用"损失厌恶"策略时：**
- 公式：[对象] + [触发动词] + [受害者]
- 示例："AI在骗你"、"你在漏钱"

**当使用"信任转移"策略时：**
- 公式：[权威] + [动作] + [对象]
- 示例："吴恩达力荐"、"官方终于出手"

**当使用"好奇驱动"策略时：**
- 公式：[对象] + [反常识动词] + [悬念]
- 示例："AI幻觉是故意的"、"GPT会撒谎"

**当使用"时间压缩"策略（工具教程类）时：**
- 公式：[工具名/主题]
- 示例："Agent Skills"、"Cursor教程"

**当使用"资源推荐"策略时（基于对标博主爆款规律）：**
- 公式：[数字]个[资源类型]
- 示例："5个必看资源"、"10个开源项目"
- 特点：数字化+资源类型，收藏比极高（1.8-2.3）

**当使用"时间承诺+降低门槛"策略时（基于对标博主爆款规律）：**
- 公式：[时间]+[降低门槛词]
- 示例："10分钟讲透"、"3分钟上手"、"小白也能学"
- 特点：时间承诺+降低门槛，涨粉率高（2-4%）

---

**副标题（第二行）：**

**字数要求：** 3-8字

**结构公式（与标题互补）：**

**当使用"损失厌恶"策略时：**
- 如果标题有时间 → 副标题说结果承诺
- 如果标题无时间 → 副标题说时间+动词

**当使用"信任转移"策略时：**
- 如果标题有权威 → 副标题说价值承诺
- 如果标题无权威 → 副标题说时效性+价值

**当使用"时间压缩"策略（工具教程类）时：**
- 如果标题有时间 → 副标题说核心价值
- 如果标题无时间 → 副标题说时间+身份

## 标题生成规则

> ⚠️ 重要：小红书标题禁用｜分隔符，用逗号/括号/感叹号代替

> ⚠️ 硬限制：**标题不超过20字**。英文按字符数计（deadline=8字）。生成后必须逐条数字数，超过20字不得输出，裁剪后再输出。

**结构公式：**
```
[情绪钩子3-5字] + [具体场景5-8字] + [身份标签3-5字] + [结果暗示2-3字]
```

## 标题语气检测（生成后必须执行）

**以下特征是XHS失败标题的信号，生成后必须检查：**

| ❌ 失败特征 | 原因 | ✅ 替换方向 |
|---|---|---|
| 使用｜分隔符 | B站/公众号风格，XHS违和 | 改用逗号、括号、感叹号 |
| 判断式开头："X是错的/不是X是Y" | 说教感，制造距离 | 改为口语化问句或第一人称 |
| 模糊承诺动词："彻底解决/完全搞定" | 广告感，不可信 | 改为具体数字或叙事结果 |
| "99%的人不知道" | XHS用户反感被划等级 | 改为第一人称发现式 |
| 无时间承诺/无具体数字（工具类） | 缺乏多巴胺预期触发 | 加入"X分钟"时间承诺 |

## 承诺强度校验（内容匹配度检查）

**根据内容实际交付确定承诺上限：**

| 内容实际交付 | 允许使用 | ❌ 禁止使用 |
|---|---|---|
| 展示工具/演示功能 | "实测/上手/搞定" | "彻底解决/完全搞定" |
| 分享个人经历 | "发现/找到/变了/改善了" | "保证有效/100%成功" |
| 有前后对比数据 | "提升X%/完成率翻倍" | 无数据支撑时用此类 |
| 纯教程演示 | "教你/手把手/保姆级" | "3秒学会/秒懂" |
| 方法论分享（无数据） | "我的方法/我在用的" | "终极方案/彻底解决" |

**校验三问（生成后必问）：**
1. 内容里有没有前后对比数据？→ 没有则不能说"翻倍/提升X%"
2. 内容是展示工具还是证明效果？→ 展示则不能说"彻底解决"
3. 标题承诺的，内容里有没有直接兑现？→ 没有则降低承诺强度

## 封面与标题的互补关系

**核心原则：封面与标题的互补关系**

**封面配文的作用：**
- 在0.5秒内抓住眼球
- 传递最核心的情绪钩子或主题
- 字数控制在3-8字（极简）
- **禁止重复标题中的时间/数字信息**

**标题的作用：**
- 在3秒内说清楚完整价值
- 补充封面没说的细节、身份标签、结果承诺
- 字数控制在20字以内
- 包含封面没说的信息

**正确关系：**
- 封面 + 标题 = 完整信息架构（分工互补，而非机械无重叠）

**关键区分：信息架构 vs 真正冗余**

**✅ 分工互补（允许）：**
- 封面"OpenClaw" + 封面"飞书版" + 封面"0基础极速接入" → 三行各司其职（产品名/变体/门槛承诺），观众0.5秒能解码完整画面
- 封面"0基础" + 标题"小白入门" → 分别击中能力焦虑和身份认同（见武器8），不是重复

**❌ 真正冗余（需要调整）：**
- 封面有"3分钟"+ 标题也有"3分钟" → 完全相同信息，零增量
- 封面"极简安装" + 标题"简单上手" → 同一维度重复，无新信息

**判断标准：** 不是问"有无重叠字词"，而是问"两处表达是否触发了不同的认知/情绪维度，以及观众能否在0.5秒内解码"
