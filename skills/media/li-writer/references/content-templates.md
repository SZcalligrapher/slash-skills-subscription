# 内容类型模板

本文档包含各类内容的完整模板和示例。

---

## 模板1：经验分享型

```markdown
【开场】今天分享一下[主题]

【背景】最近/上周[具体情境]

【经历】[详细叙述过程，包含具体数据]

【体会】通过这次实践，我最大的体会是：[核心观点]

【方法】我把整个过程总结为X步：
第一步：[具体步骤]
第二步：[具体步骤]
第三步：[具体步骤]

【类比】就像[生活化比喻]

【结尾】希望对你有帮助。如果你也对[主题]感兴趣，欢迎关注我！
```

---

## 模板2：教程型

```markdown
【开场】经常有人问我[问题]，今天就来分享一下[解决方案]

【问题】[描述普遍痛点，建立共鸣]

【方案】我的方法是[核心方法]

【步骤】具体来说：
1. [步骤1 + 具体操作]
2. [步骤2 + 具体操作]
3. [步骤3 + 具体操作]

【建议】我建议：[针对性建议]

【类比】[生活化比喻降低理解门槛]

【结尾】以上就是[主题]，希望对你有帮助！
```

## 模板3：养成记型（周更vlog）

```markdown
【开场】大家好，我是鲤哥，一个目标成为"一人公司"的独立开发者！今天是我记录独立开发的第X周，和大家分享一下我这周的成果～

【引入】上周/上周我[引出本周干的事情]...

【展开】[详细叙述过程，包含数据和细节]

【反思】通过这次[事件]，我最大的体会是：[核心观点]

【数据】[具体的数字：收入、流量、下载量等]

【心得】[提炼出的方法或建议]

【结尾】ok，如果你对独立开发感兴趣，或者想围观我是如何一步步打造"一人公司"的，欢迎关注我！也欢迎在评论区交流你的想法，下周再见！拜拜！
```

**注意事项：**
- 内容要体现成长、价值和未来可期的成长性
- 保持真实性，适当暴露不完美和困境

## 模板4：资源推荐型（基于对标博主爆款规律）

```markdown
【开场】学习[主题]最重要的是找对资源

【痛点】网上资源太多，不知道看哪个，我之前也踩过很多坑

【筛选标准】我的筛选标准是：[3个标准]
比如：实战性强、更新及时、社区活跃

【资源清单】我筛选出了X个资源：

第一个：[资源名]
- 核心价值：[一句话说明]
- 适用场景：[谁适合用]
- 我的体会：[真实使用感受]

第二个：[资源名]
- 核心价值：[一句话说明]
- 适用场景：[谁适合用]
- 我的体会：[真实使用感受]

...

【学习建议】如何使用这些资源：
- 建议1：[具体建议]
- 建议2：[具体建议]

【结尾】以上就是我筛选的X个[主题]资源，希望对你有帮助！评论区扣1，我发给你完整清单。
```

**注意事项：**
- 资源推荐型内容收藏比极高（1.8-2.3），用户会反复查看
- 必须有真实使用体验，不能只是罗列
- 每个资源都要说明适用场景，帮用户做选择
- 标题公式："自学[主题]，一定要看的X个[资源]" / "X个开源[主题]项目推荐"

## 模板5：热点事件解读型（基于对标博主爆款规律）

> 数据支撑：黑猩猩「51万行源码泄露」11.6万观看；蜗牛「苹果刚刚发布了4599元的MacBook」170万观看、48162赞

```markdown
【开场】[时间点]，[事件主体][震撼数字的事件描述]。[一句话说明为什么反常/重要]。

【背景】[事件的来龙去脉，快速交代]

【拆解1】[第一个衍生项目/影响]
- [具体做了什么]
- [为什么重要]

【拆解2】[第二个衍生项目/影响]
- [具体做了什么]
- [与第一个的差异]

【用户痛点】[直击目标用户的具体痛点，如封号、价格、使用限制]

【反差叙事】[官方说法 vs 实际真相，制造反差感]

【总结】[一句话观点收尾，不说教]
```

**注意事项：**
- 收藏/点赞比通常<1.0，属于情绪传播型内容，目标是拉高观看量和曝光
- 涨粉率偏低（蜗牛MacBook 0.04%），适合冲播放量而非涨精准粉
- 数字必须具体且震撼（"51万行"而非"大量代码"）
- 时效性是核心——事件发生当天出视频效果最好
- 标题公式：`[震撼数字+事件] + [破纪录/极端结果]`

## 示例输出

### 示例1：经验分享型

**用户请求**：写一篇关于如何用AI提效的内容

**输出**：

```
今天分享一下我是如何用AI提升开发效率的。

最近我在开发一个新项目，发现自己花了大量时间在写重复代码上。作为独立开发者，时间就是最宝贵的资源，所以我开始尝试用AI来提效。

通过这次实践，我最大的体会是：AI不是用来替代你思考的，而是用来替代你执行的。

我把整个AI提效的过程总结为三步：

第一步，明确你要做什么。在让AI写代码之前，我会先用纸笔画出功能流程图，把逻辑理清楚。这一步AI帮不了你，必须自己思考。

第二步，给AI提供足够的上下文。我会把相关的代码文件、功能说明、甚至之前的bug记录都喂给AI，让它理解整个项目的背景。

第三步，迭代优化。AI第一次生成的代码通常不完美，我会运行测试，把报错信息再反馈给AI，让它修复。通常3-5轮迭代就能得到可用的代码。

用这个方法，我的开发效率提升了大概3倍。以前要花一天写的功能，现在3-4小时就能搞定。

就像你不会做饭，但可以用外卖解决吃饭问题。AI就是开发者的"外卖"，关键是你要知道自己想吃什么。

希望对你有帮助。如果你也在用AI开发，欢迎在评论区分享你的经验！
```
