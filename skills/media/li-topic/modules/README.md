# li-topic 模块化使用指南

## 📁 模块结构

```
li-topic/
├── SKILL.md (主入口)
└── modules/
    ├── 1-topic-analyzer.md (选题诊断)
    ├── 2-outline-designer.md (大纲设计)
    └── 3-style-validator.md (风格校验)
```

## 🔄 使用流程

### 完整流程(推荐)
```
选题 → 模块1(诊断) → 模块2(大纲) → 模块3(风格校验) → 输出
```

### 灵活组合
- 只做诊断: 模块1
- 只做大纲: 模块2
- 只做风格校验: 模块3
- 诊断+大纲: 模块1+2
- 大纲+风格校验: 模块2+3

## 📝 调用方式

### 方式1: 完整流程(默认)
```
/li-topic [选题内容]
```
自动执行: 模块1 → 模块2 → 模块3

### 方式2: 指定模块
```
/li-topic --module=1 [选题内容]  # 只做诊断
/li-topic --module=2 [选题内容]  # 只做大纲
/li-topic --module=3 [大纲内容]  # 只做风格校验
```

### 方式3: 组合模块
```
/li-topic --modules=1,2 [选题内容]  # 诊断+大纲
/li-topic --modules=2,3 [选题内容]  # 大纲+风格校验
```

## 🎯 各模块输出

### 模块1: 选题诊断
- 选题层级判定(🚲/🛵/🚗/✈️)
- 核心问题分析
- 保守/激进升级方案
- 流量预估

### 模块2: 大纲设计
- 分段结构(5段)
- 情绪波峰设计
- 时间节点规划
- 文案框架(非完整文案)

### 模块3: 风格校验
- 风格匹配度评分
- 不符合风格的表达
- 具体改写建议
- 基于 li-writer 文风 DNA

## 💡 使用建议

1. **首次创作**: 使用完整流程(模块1+2+3)
2. **快速迭代**: 只用模块2(大纲设计)
3. **风格检查**: 只用模块3(风格校验)
4. **选题评估**: 只用模块1(选题诊断)

## 🔧 与其他skill的配合

```
工作流建议:
1. li-topic (模块1+2) - 诊断+大纲
2. li-topic (模块3) - 风格校验
3. li-writer - 生成完整脚本
4. li-cover - 生成封面配文
```

## 📊 模块依赖关系

```
模块1 (选题诊断)
  ↓ 可选
模块2 (大纲设计) ← 可以独立使用
  ↓ 必需
模块3 (风格校验) ← 需要大纲/文案作为输入
```

## ⚠️ 注意事项

1. 模块3(风格校验)需要先有大纲或文案
2. 模块1和模块2可以独立使用
3. 完整流程会自动执行所有模块
4. 每个模块的输出都会保存,方便后续调用
