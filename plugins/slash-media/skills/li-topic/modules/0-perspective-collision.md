# Module 0: 视角碰撞

**目标**：在认知劫持分析之前，先从创作者的真实视角确定选题角度。避免从模板推导出"像创作者的内容"，而是从他的真实立场出发找到角度。

---

## 执行步骤

### 第一步：读取创作者文风 DNA

读取 `.claude/skills/li-writer/SKILL.md`，提取以下核心特征：

- **哲学框架**：经验主义（知识来自第一人称实践）、平民主义（反权威、平等对话）、透明化（Build in Public）
- **核心身份**：独立开发者 + AI 博主 + 前后端工程师 + 一人公司实践者
- **表达方式**：具体数据 > 抽象观点，"分享"而非"教导"，暴露不完美

---

### 第二步：三个碰撞问题

基于以上特征，对当前选题依次回答：

**问题 1 — 创作者视角**
> 他对这个话题有没有真实经历或立场？如果让他来说这件事，他的第一反应会是什么？他会从哪个具体场景切入，而不是从功能介绍切入？

判断标准：是否能用"我上周/上个月..." 开头说出来？如果不能，这个角度对他来说是空的。

**问题 2 — 挑战者视角**
> 这个选题最显而易见的角度是什么？为什么用户可能会刷走（已经看过太多同类内容）？有没有一个更有张力的问题被第一反应忽略了？

判断标准：如果把这个选题给另外 10 个 AI 博主，他们会做出什么内容？创作者的角度和他们有什么不同？

**问题 3 — 受众视角**
> 点开这条视频的人，他们表面上想要什么？背后真正在意的是什么（身份认同 / 焦虑消解 / 能力获取）？

判断标准：如果只解决表面需求，用户会收藏但不会关注；如果触达背后动机，用户会关注并转发。

---

### 第三步：输出精炼角度建议

三个问题碰撞后，用以下格式输出，控制在一屏以内：

```
## 视角碰撞结果

**创作者的真实切入点**：[一句话，具体到某个场景或经历]

**被忽略的有张力问题**：[一句话，说明显而易见的角度有什么问题，更好的角度是什么]

**受众背后的真实动机**：[一句话]

**精炼角度建议**：
[1-2 句话，综合以上三点，给出一个具体的选题角度。这个角度应该同时满足：创作者能真诚说出口 + 和同类内容有差异 + 触达受众真实动机]

---
进入 Module 1 认知劫持分析 →
```

---

## 注意事项

- 如果创作者对这个话题**没有真实经历**，在创作者视角里直接说明，并建议他先试用/测试再做内容，或者换一个他有真实立场的角度
- 如果是**商单内容**，创作者视角的问题变成：他对这个产品有没有真实的观察或判断？哪个功能对他的工作流有实际价值？（不能只从 brief 推导）
- Module 0 是**角度确认**，不是大纲设计。输出的是方向，不是结构
