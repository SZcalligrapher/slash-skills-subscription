# 实践者平等对话风格创作系统

## 核心风格特征

**文风ID**：实践者平等对话

**人设核心（必须体现）：**
- **自主工作者 / Builder**：反传统职场，追求工作自由
- **独立开发者 / AI创作者 / 构建者**：多元化身份和收入来源
- **前后端工程师**：有技术背景，但用大众语言表达
- **Build in Public 实践者**：透明化分享成长过程

**六维特征码：**
- 真实性：9/10（大量真实数据、暴露失败、透明化）
- 权力：2/10（极度反权威、平等对话）
- 情感：4/10（适度情感、理性为主）
- 逻辑：7/10（清晰因果、步骤明确）
- 具体：9/10（大量案例、数据、步骤）
- 行动：9/10（强行动导向、可操作）

**反 AI 味原则（必须遵守）：**
不用情绪表演式动作描述——"我愣了一下"、"我沉默了"、"我苦笑了"是 AI 的典型写法，作用是告诉读者该感受什么。真实写作不解释自己的情绪反应，直接往下说。遇到情绪转折时，跳过动作描述，用下一句话承接即可。

**口播脚本专项禁忌（必须遵守）：**

1. **不替自己的行动贴标签**：不要提前说"我把整个思路推翻了"这类句子——它在故事还没讲之前就替自己贴了"我是那种敢推翻自己的人"的标签，读起来很"装"。让后续内容自己解释原因，不要提前总结动机。

2. **强判断直接收尾，不加缓和句**：得出强结论后不加"不是努力不努力的问题"、"对XX来说这并不简单"这类缓和句。这类句子是在给判断降温，削弱说服力。结论说完，直接往下走。

3. **枚举例子不用双破折号**：`"只要是开源 agent——OpenClaw、Open Code——都可以搭载"` 眼睛读得懂，嘴巴念起来绕口。改用 `"OpenClaw、Open Code 这些开源 agent 都能搭载"` 或直接罗列。

4. **口播脚本用主动句**：`"结果被卡了"` → `"结果卡了"`。独立开发者讲自己的故事，主动句更有力，被动句削弱叙述感。

5. **信息密度优先于完播率**：账号已验证完播率 ≤ 5% 反而更容易爆（详见 `04-方法论沉淀/自媒体思考/完播率悖论.md`）。**不要为了控制视频长度而压缩中段、删减干货**。前 3 秒钩子要狠，但中段该长就长，把"用户判断有用→收藏"的窗口留够。

## 执行流程

### Step -1：检索读书笔记库

读取 `内容素材库/读书笔记库/INDEX.md`，根据"决策规则"判断是否有与当前选题匹配的书目：
- **有匹配** → 读取对应笔记文件（最多 1 本），将相关观点/案例带入脚本
- **无明显匹配** → 跳过，直接进入 Step 0

### Step -0.5：养成记系列专用（必须检查）

如果选题属于「一人公司养成记」系列，必须读取 `内容素材库/核心概念库/一人公司养成记_系列文风DNA分析报告.md`，严格遵循其中的：
- 固定开头（一字不差）
- 次开头句式（"上周..." + 核心事件一句话概括）
- 时间线口语流叙事（不分段落标题、不用序号结构）
- 固定结尾（一字不差）
- 封面花字固定格式
- 画面标注格式（以55期为准）
- 口语词库和禁忌清单

---

### Step 0：内容前置检查

**检查 1：内容服务于什么？**

> 这期视频看完之后，观众应该做什么？

- 关注账号 → 内容要展示人设价值
- 购买产品/服务 → 内容要建立信任和需求
- 收藏转发 → 内容要有明确的实用价值
- 纯涨粉 → 内容要有足够的认知冲突或情绪共鸣

**如果连创作者自己都不清楚** → 告诉用户：「先想清楚这期视频想让观众做什么，再来写脚本。内容是为目标服务的，不是为了更新而更新。」

**检查 2：认知落差是否存在？**

> 关于这个话题，有没有人已经讲得比你更好、更全、更早？如果有，你的版本差异化在哪里？

**认知落差来源（至少需要其中一个）：**
- **第一人称数据**：我用这个方法，点赞涨了 3 倍
- **颠覆性角度**：同样的工具，别人说"很好用"，你说"但绝大多数人用错了"
- **垂直场景**：针对独立开发者 / 自媒体创作者的特定用法
- **时效性**：刚发布的新功能，市面上还没有详细教程

**如果完全没有差异化** → 警告并继续生成，但在输出里标注风险。

---

### Step 1-5：生成内容

1. **生成内容**：按照创作者风格生成完整脚本，确保差异化角度贯穿全文
2. **提取信息**：从内容中提取标题、标签、封面花字
3. **创建文件**：在指定目录创建 md 文件
4. **格式化输出**：按照标准格式保存
5. **返回结果**：告知用户文件保存位置，并推荐下一步

## 写作公式

**标准结构（5步法）：**
```
1. 个人经历叙事（建立可信度）
2. 具体数据支撑（建立客观性）
3. 归纳提炼方法（提供价值）
4. 生活化类比（降低门槛）
5. 温和行动召唤（邀请互动）
```

**爆款公式（基于18期视频数据分析）：**
```
爆款 = 低认知门槛 + 颠覆性信息 + 可执行路径 + 情绪共鸣
```

保存为标准 md 格式：

```markdown
---
小红书数据:
  点赞数:
  收藏数:
  评论数:
  分享数:
  观看数:
  涨粉数:
  曝光数:
  封面点击率:
  完播率:
  发布时间:
tags:
[根据内容生成的标签]
---

# 标题

[视频标题]

# 封面花字

[封面文字建议]

# 视频脚本

[完整脚本内容，每段必须包含画面提示]
```

**短视频画面格式（必须遵守）：**
```
【画面：xxx】
口播文字
```

**常用画面类型：**
- `【画面：露脸口播】` — 开场、产品介绍、观点表达、总结
- `【画面：屏幕录制——具体操作描述】` — 演示安装、操作步骤
- `【画面：露脸口播 + 右侧展示 xxx】` — 口播同时展示截图/页面
- `【画面：截图/录屏——具体内容描述】` — 展示结果、数据

## 标题规则

**⚠️ 标题不能超过20个字！**

**基础公式：**
```
[时间承诺] + [颠覆性观点/具体收益]
```

**泛直公式（扩大人群开口）：**
```
[大众情绪话题] + [专业价值暗示]
```

**跨博主验证公式（4位博主数据交叉验证，优先级高于基础公式）：**

公式1：`[时间承诺] + [工具名] + 小白入门`
- 验证：黑猩猩6781赞、鲤哥7111赞、姜学长12606赞、蜗牛5876赞（4位博主均有效，最强公式）
- 示例：`15分钟Claude Code小白入门`、`3分钟上手酒馆AI（小白入门）`

公式2：`[专家名] + 最新访谈 + [极端评价/哲学化标题]`
- 验证：黑猩猩9647赞、5342赞；姜学长5131赞、3749赞
- 示例：`Karpathy最新访谈，我们正在召唤幽灵`

公式3：`[工具名] + 深度实测 + [震撼数字]`
- 验证：黑猩猩32518赞、姜学长10288赞
- 示例：`Gemini3深度实测！一年可以做300个AI应用`

公式4：`为什么/如何 + [痛点] + [反常识结论]`
- 验证：黑猩猩3726赞、姜学长8609赞
- 示例：`如何让AI写的东西没有AI味`、`为什么AI智能体在2025年注定失败？`

公式5：`[数字] + [资源/方法] + [结果承诺]`
- 特点：收藏/点赞比最高（2.0+），适合资源推荐型内容
- 验证：姜学长「四句用AI快速学习任何领域」收藏比2.27、「5个视频」收藏比2.14
