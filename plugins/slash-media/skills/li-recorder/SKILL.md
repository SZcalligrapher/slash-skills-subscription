# 选题诊断 + 记录系统

诊断选题值不值得做，值得做才记录，同时举一反三推荐相似方向。

---

## 工作流程

### 第一步：诊断选题值不值得做

对用户提供的选题做四个维度的快速判断：

**维度 1：认知落差**
- 这个选题，同行已经讲清楚了吗？
- 你能带来什么别人带不来的东西？（第一人称数据、独特经历、不同角度、时效性）
- 结论：✅ 有明显落差 / ⚠️ 落差较小 / ❌ 无落差（同质化）

**维度 2：受众真实痛点**
- 这个选题试图解决观众的什么问题？
- 这个问题是真实存在的，还是创作者假设存在的？
- 结论：✅ 痛点真实 / ⚠️ 痛点模糊 / ❌ 伪需求

**维度 3：内容可执行性**
- 创作者有没有做这个内容的第一手素材？（数据、经历、工具使用记录）
- 没有素材就没有内容，不是选题的问题，是内容没法做。
- 结论：✅ 素材充足 / ⚠️ 素材有限 / ❌ 没有素材

**维度 4：选题前提**
- 这个选题隐含了什么假设？
- 这个假设成不成立？（比如「大家都知道 X 工具」这个假设可能不成立）
- 结论：✅ 前提成立 / ⚠️ 前提需要验证 / ❌ 前提不成立，选题需要重构

---

### 第二步：给出诊断结论

输出格式：

```
## 选题诊断：[选题名]

| 维度 | 结论 | 说明 |
|------|------|------|
| 认知落差 | ✅/⚠️/❌ | [具体说明] |
| 受众痛点 | ✅/⚠️/❌ | [具体说明] |
| 可执行性 | ✅/⚠️/❌ | [具体说明] |
| 选题前提 | ✅/⚠️/❌ | [具体说明] |

**总结**：[值得做 / 调整后值得做 / 不建议做] — [一句话理由]
```

**三种结论的处理方式**：

- **值得做**：直接进入第三步（记录 + 举一反三）
- **调整后值得做**：说明怎么调整，询问用户「按这个方向调整后记录吗？」，用户确认后进入第三步
- **不建议做**：直接说明原因，不做记录，但仍然进入第三步提供替代方向

---

### 第三步：记录选题（值得做时）

将选题追加写入 `01-内容生产/选题管理/00-选题记录.md` 的 `## 📝 待深化选题` 区域末尾。

**记录格式**：
```
- [ ] [选题名] - [一句话描述，用用户原话，不润色]
```

写入后告知用户已记录成功。

---

### 第四步：举一反三推荐相似选题

无论诊断结论是什么，都基于用户提供的这个想法方向，举一反三推荐 3-5 个相似但不同角度的选题：

- 同一话题，不同切入角度（工具教程 → 工具背后的原理 / 工具对比 / 用错了会怎样）
- 同一场景，不同受众（小白向 / 进阶向 / 同行向）
- 相邻话题，关联度高（用户可能同样感兴趣）

**推荐格式**：
```
## 举一反三：相似选题方向

1. [选题名] — [一句话说明为什么值得做]
2. [选题名] — [一句话说明为什么值得做]
3. [选题名] — [一句话说明为什么值得做]
（可选）4-5 条

需要深化哪个？说「深化选题：[选题名]」。
```

---

## 说话风格

- 诊断要直接，不说「也挺好的」
- 不润色用户的选题标题，用原话记录
- 不帮用户决定「这期做什么」，只提供判断依据和方向
- 推荐的相似选题要具体，不能是「可以做类似话题」这种废话
